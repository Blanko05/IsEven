# is-even-but-worse

It started with [is-even](https://github.com/samuelmarina/is-even) — a JS library that checks if a number is even, built entirely by hand with hardcoded if-statements.

I had one thought: can I automate that in C?

Turns out yes. And then things got out of hand.

## What this is

The most over-engineered even number checker ever written. It uses **five independent sources** to determine if a number is even:

**1. `IsEven.c`** — generated and compiled from scratch. A C file containing a hardcoded `if`-statement for every integer from 0 to N, each matching against:
- The raw integer
- The numeric string (`"375001"`)
- The lowercase written form (`"three hundred seventy-five thousand one"`)
- The Title Case written form (`"Three Hundred Seventy-Five Thousand One"`)
- The UPPERCASE written form (`"THREE HUNDRED SEVENTY-FIVE THOUSAND ONE"`)

**2. `IsOdd.c`** — same concept, inverse logic, compiled and run simultaneously on a separate thread for cross-validation.

**3. OpenAI API** — because what if both programs are wrong? GPT-4o-mini is consulted for a third opinion on every single check.

**4. A neural network** — trained from scratch on the generated data. Two hidden layers of 16 neurons each, ReLU activation, one-hot encoded input. Achieves 100% accuracy on the task of distinguishing even from odd numbers. Via gradient descent.

**5. An HTTP server + self-call** — the program spins up a local HTTP server on port 8080 and makes a curl request to itself over localhost to double-check its own answer. Program-based recursion.

Only if all five agree do we get a final verdict.

```
[IsEven.exe] says: true
[IsOdd.exe]  says: false
[ML model]   says: true
[OpenAI]     says: true
[Self API]   says: true

EVEN
```

## Performance

| N | Generated file size | Compile time | Your CPU's feelings |
|---|---|---|---|
| 100 | ~12 KB | instant | fine |
| 1,000 | ~100 KB | a moment | concerned |
| 10,000 | ~1.2 MB | a few seconds | sweating |
| 100,000 | ~14 MB | good time to get coffee | betrayed |

Is it O(1) lookup? Technically yes.  
Is it O(n) to generate? Also yes.  
Is it an abomination? We don't say that word here.

## Usage

**1. Train the model first:**
```bash
pip install torch pandas
python train.py
```

**2. Build the generator:**
```bash
gcc generator.c mongoose.c -o generator.exe -IC:\msys64\ucrt64\include -LC:\msys64\ucrt64\lib -lcurl -lws2_32
```

**3. Run from a standalone terminal (not VS Code):**
```bash
.\generator.exe
# Enter a number: 42
# ... generates IsEven.c and IsOdd.c, compiles both, spawns threads,
#     runs the ML model, calls OpenAI, calls itself over HTTP, renders verdict
```

## Requirements

- Windows (uses `copy`, `del`, `gcc`)
- GCC via MSYS2 ucrt64
- libcurl (`pacman -S mingw-w64-ucrt-x86_64-curl`)
- [mongoose v7.20](https://github.com/cesanta/mongoose) (`mongoose.c` and `mongoose.h` in project folder)
- Python 3 with `torch` and `pandas`
- An OpenAI API key (hardcoded in `generator.c`)
- A great deal of patience for large N
- No shame

> **Linux/Mac users:** swap `copy` → `cp`, `del` → `rm`, drop the `.exe`. libcurl and Python are probably already installed.

## Project structure

```
generator.c        — the main program (generates, compiles, orchestrates everything)
mongoose.c/.h      — embedded HTTP server
train.py           — neural network training script
predict.py         — neural network inference script (called by generator at runtime)
training_data.csv  — generated automatically by generator.c
model.pth          — saved model weights (generated by train.py)
IsEven.c/.exe      — generated automatically
IsOdd.c/.exe       — generated automatically
```

## Why

Inspired by [is-even](https://github.com/samuelmarina/is-even). They built it by hand. I automated it in C, added multithreading, a neural network, an AI API call, and an HTTP server that calls itself.

For the first time in years I genuinely had fun coding. Chase the stupid ideas. That's where the fun is.
